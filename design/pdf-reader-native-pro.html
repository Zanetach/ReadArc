Chapter 03 · Capacity Planning

Local AI Resource Planning and Deployment Boundaries

This page demonstrates the reader target state: precise PDF rendering, contextual search, persistent navigation, and a quiet inspector that does not compete with the document.

The recommended deployment starts with a small master node and expands worker capacity only when measured throughput requires it. The document view should keep the original page dominant and expose operational controls at the edge of attention.

For repeated review work, resource planning terms should be easy to locate without replacing the PDF page with a separate search screen. Search is treated as a lightweight layer over the rendered page, paired with match navigation in the toolbar.

Design target: page reading stays stable while the sidebar, inspector, and search state behave like native macOS utilities around the document.

The first implementation can ship with recent files, page navigation, zoom, and match jumping. The next layer adds outline extraction, metadata, annotations, and document comparison.
